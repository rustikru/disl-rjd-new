# Стиль кода — ApiController.php

## Именование переменных по роли в SQL-запросе

Переменные называются по тому SQL-блоку, в котором они используются:

| Переменная     | Роль                                                        |
|----------------|-------------------------------------------------------------|
| `$source`      | Массив `['from' => '...', 'bindings' => [...]]`             |
| `$rowDims`     | Массив полей GROUP BY / измерений строк                     |
| `$colDefs`     | Массив определений колонок-измерений                        |
| `$selectRow`   | Строка для SELECT: поля строк (`implode(', ', $rowDims)`)   |
| `$selectCols`  | Строка для SELECT: вычисляемые колонки                      |
| `$groupByCols` | Строка для GROUP BY: выражения колонок                      |
| `$colAliases`  | Массив алиасов колонок                                      |
| `$orderByCols` | Строка для ORDER BY внутри `summaryReport()`                |
| `$whereCond`   | Строка условия WHERE (начинается с ` AND ...`)              |
| `$orderBy`     | Финальная строка ORDER BY в detail-хендлерах                |
| `$bindings`    | Массив bind-параметров для запроса                          |
| `$selectCols`  | Строка полей SELECT в detail-хендлерах                      |

Правило: **имя переменной отражает её место в SQL-конструкции**, а не абстрактную роль ("base", "gf", "where", "select").

## Единый стиль Summary и Detail хендлеров

### Summary (шаблон)
```php
$params  = $request->getQueryParams();
$source  = $this->xyzFrom($params);
if (!$source['reportDt']) {
    return $this->json($response, ['cols' => [], 'roads' => [], 'metrics' => [], 'total' => 0]);
}
$rowDims = $this->parseGroupBy($params['group_by'] ?? '', ['default_field']);
$colDefs = $this->resolveColDims($params['col_by'] ?? '', ['allowed_alias']);
return $this->json($response, $this->summaryReport($source, $rowDims, $colDefs));
```

### Detail (шаблон)
```php
$params    = $request->getQueryParams();
$rowDims   = $this->parseGroupBy($params['group_by'] ?? '', ['default_field']);
$source    = $this->xyzFrom($params);
if (!$source['reportDt']) {
    return $this->json($response, ['rows' => []]);
}
$bindings  = $source['bindings'];
$whereCond = $this->applyDetailFilters($rowDims, $params, $bindings);
$selectCols = $this->selectFields($params['fields'] ?? '');
$orderBy   = $this->orderBY($params, implode(', ', $rowDims));
$rows = $this->db->fetchAll(
    "SELECT $selectCols FROM {$source['from']} WHERE 1=1 $whereCond ORDER BY $orderBy",
    $bindings
);
return $this->json($response, ['rows' => $rows]);
```

## Ключевые методы

- `parseGroupBy($str, $defaults)` — парсит поля GROUP BY, валидирует, возвращает массив
- `summaryReport($source, $rowDims, $colDefs)` — единый построитель сводного запроса
- `applyDetailFilters($rowDims, $params, &$bindings)` — единый построитель WHERE: строковые фильтры (prefix `row_`) + колонковые фильтры из `colDimRegistry()` (prefix `col_`)
- `colDimRegistry()` — единый источник истины для вычисляемых измерений колонок (`wagon_type_code`, `cargo_w_type`)
- `orderBY($params, $default)` — строит ORDER BY из параметров sort/sort_dir/sort_type
- `resolveColDims($colBy, $allowed)` — возвращает `$colDefs` по запрошенному алиасу из `colDimRegistry()`

---

## Что запрещено

- Разные имена для одного и того же по смыслу (`$base` vs `$source`, `$gf` vs `$rowDims`, `$outerWhere` vs `$whereCond`) — **не допускается**
- Дублирование тела `summaryReport()` внутри конкретных хендлеров — **не допускается**
- Отдельный метод `applyGfFilters` — **устранён**, логика встроена в `applyDetailFilters`
- Непоследовательное именование bind-параметров (`gfv_*` → теперь `row_*`, колонки → `col_*`)

## Конфликты при push

Если при `git push` возникает non-fast-forward:
1. `git fetch origin <branch>`
2. `git rebase origin/<branch>`
3. При множественных конфликтах из-за глобального переименования: `git rebase --abort`, сохранить чистую версию из `git show <commit>:path/file`, сбросить на `origin`, наложить поверх.
