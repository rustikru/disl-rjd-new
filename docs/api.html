API Reference

Внутренний REST API системы дислокации вагонов.
Все эндпоинты требуют активной сессии (cookie). Данные возвращаются в формате JSON. Базовый путь зависит от конфигурации сервера (base_path).

Введение

API работает поверх сессионной аутентификации. Все запросы к /api/* должны содержать действующий session cookie, полученный через форму входа POST /login. Доступ к отдельным разделам ограничен ролями пользователя.

Общие параметры (применимы к большинству эндпоинтов)
ПараметрТипОписание
report_dtstringДата справки YYYY-MM-DD. Если не указана — берётся последняя загруженная.
cargostringФильтр по наименованию груза (точное совпадение, без учёта регистра и буквы Ё/Е).
wagon_nostringНомер(а) вагона через точку с запятой: 12345678;87654321.
group_bystringПоле(а) группировки строк сводной таблицы через запятую.
col_bystringПоле(а) группировки колонок сводной таблицы через запятую.
fieldsstringСписок полей для возврата в подробной таблице через запятую.
sort_bystringПоле сортировки подробной таблицы.
sort_dirstringasc или desc (по умолчанию asc).
Формат ответа об ошибке
JSON
{
  "status": "error",
  "message": "Недостаточно прав для доступа к разделу"
}
Аутентификация
POST /login

Вход в систему. При успехе устанавливает session cookie.

Тело запроса (application/x-www-form-urlencoded)
ПараметрОбязательныйОписание
usernameдаЛогин пользователя
passwordдаПароль
csrf_tokenдаCSRF-токен из формы
Запрос
# При успехе — редирект 302 на /
# При ошибке  — редирект 302 на /login?error=...
curl -c cookies.txt -d "username=admin&password=secret&csrf_token=TOKEN" /login

KPI карточки
GET /api/kpi/summary

Возвращает набор KPI-карточек с текущим значением и трендом относительно предыдущей справки. Значения вычисляются Oracle-пакетом xx_rjd_dislocation_new_pkg.

🔒 Требует роль dashboard
Query-параметры
ПараметрОбязательныйОписание
kpi_typeнетФильтр по типу карточки (например, wagon, cargo).
Запрос
GET /api/kpi/summary
GET /api/kpi/summary?kpi_type=wagon
Пример ответа
{
  "updated_at": "20.06.2026 16:01",
  "sections": [
    {
      "values": [
        {
          "id": "1",
          "label": "Всего вагонов",
          "value": "1 842",
          "trend": "up",
          "change": "+2.3%"

          "id": "2",
          "label": "Гружёных",

    }
  ]
}

Сводная таблица дислокации. Объединяет данные справок «Подход» и «Отправка».

Query-параметры
ПараметрОбязательныйОписание

cargoнетФильтр по грузу.
wagon_noнетНомер(а) вагона через ;.

GET /api/dislocation/summary?report_dt=2026-06-20&cargo=Метанол&group_by=dest_road

  ],
  "roads": ["СВЕРДЛОВСКАЯ", "ПЕРМСКАЯ"],
  "metrics": [
    {
      "dest_state": "РФ",
      "dest_road": "СВЕРДЛОВСКАЯ",

Импорт
POST /api/import/file

Загружает один XLSX-файл справки РЖД. Автоматически определяет тип справки («Подход» / «Отправка») по значению станции назначения (колонка L). При импорте удаляет предыдущую справку того же типа за тот же день и заменяет её новой.

🔒 Требует роль import
Тело запроса (multipart/form-data)
ПараметрОбязательныйОписание
fileдаXLSX-файл справки РЖД. Дата справки считывается из ячейки A2. Данные — с 5-й строки, 126 колонок.
Запрос
curl -b cookies.txt -F "file=@spravka_20062026.xlsx" /api/import/file
Успешный ответ (200)
{
  "status": "ok",
  "message": "Загружено 2000 строк",
  "rows": 2000,
  "report_dt": "20.06.2026 16:01",
  "type": "Подход"
}
Ошибка (422 / 500)
{
  "status": "error",
  "message": "Допускаются только .xlsx"
}
